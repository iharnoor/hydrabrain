hydrabrain sync

Push inHydraDB
"Why did our last enterprise deal stall — and who internally knows the full story?"
HydraDB: Each synced file becomes a memory with infer=true — people, companies, and deals link in the graph without a separate extraction pipeline. Incremental hash dedup means re-runs are cheap.
  • Sales: Sync ~/exports/crm-notes with --source sales — reps query "everything on acme-example" before calls
  • Engineering: Sync Confluence/Notion export nightly — new hires ask agents instead of hunting wikis
  • CS: Sync QBR decks and ticket summaries — account teams see cited history, not stale spreadsheets
  • Multi-team: One tenant, separate --source per department — sales never leaks into legal search
Schedule (every 15 min)
hydrabrain cron add "*/15 * * * *" "hydrabrain sync ~/company-docs --source wiki"
hydrabrain cron add "*/30 * * * *" "hydrabrain sync ~/sales-exports --source sales"

hydrabrain briefing

Push outPull digestHydraDB
"What changed across the company since yesterday — without reading 400 Slack threads?"
HydraDB: Briefing runs hybrid recall over everything your crons pushed overnight, then Gemini grounds the answer in retrieved chunks — facts come from memory, not hallucination.
  • CEO / VP: Daily 7 AM digest to email or Slack — active deals, open risks, key meetings
  • Department heads: Topic-scoped: briefing 'pipeline Q2' with --source sales
  • Board prep: Weekly briefing over --source leadership appended to a shared log
  • Async teams: APAC reads the digest file instead of attending a standup
Schedule (weekdays 7 AM)
hydrabrain cron add "0 7 * * 1-5" "hydrabrain briefing >> ~/briefings/daily-$(date +\%Y-\%m-\%d).md"
hydrabrain cron add "0 7 * * 1"   "hydrabrain briefing 'active deals' --source sales >> ~/briefings/sales-weekly.md"

hydrabrain capture

Push inHydraDB
"We just decided X in a meeting — how do we make sure every agent knows tomorrow?"
HydraDB: One-line decisions pushed via MCP capture or CLI instantly enter the graph. No embed step, no schema migration — infer=true wires entities on write.
  • Post-meeting: Agent captures "Committed to acme-example: 20% discount if signed by Q2" — searchable next morning
  • Support escalation: Capture root cause + fix — next ticket on same account gets context via think
  • Product: Capture ship/no-ship rationale — eng agents cite it in future RFC discussions
  • Cron complement: Pair with sync — bulk docs on schedule, decisions captured in real time
Agent MCP (no cron — push at decision time)
hydrabrain capture "Decision: widget-co pricing tier B for fund-a portfolio companies. Owner: sales-lead. Date: 2026-06-22."

think + MCP (agent pull)

Pull outHydraDB
"I'm on a customer call in 5 minutes — what do we know, with sources?"
HydraDB: think retrieves via hybrid recall + graph context in one call, then synthesizes with citations. Relational questions ("who at fund-a have we met who invested in competitors?") hit the native graph.
  • Sales: Cursor/Hermes agent runs think before every external meeting
  • CS: Support copilot queries account history — cited snippets, not guesswork
  • Engineering: "Why did we choose Postgres over Dynamo for service X?" — pulls RFC + incident notes
  • Onboarding: New hire's agent has MCP access to --source handbook from day one
Wire once per agent host
claude mcp add hydrabrain -- python3 -m hydrabrain.cli serve
# Agents call: think, search, graph, capture

hydrabrain export

Push outHydraDB
"Auditors want a point-in-time snapshot of what the company brain knew on date X."
HydraDB: Export dumps a namespace to Markdown + manifest.json — portable, reviewable, not locked in a proprietary format. Proves memory is auditable.
  • Compliance: Weekly cron export of --source legal to immutable storage
  • Disaster recovery: Sunday backup of full tenant before schema changes
  • M&A: Export diligence namespace when deal closes — handoff to acquirer
  • Data residency: Proof of what left the hosted store and when
Schedule (weekly)
hydrabrain cron add "0 0 * * 0" "hydrabrain export ~/brain-backups/$(date +\%Y-\%m-\%d) --source wiki"

hydrabrain read + web UI

Push inHydraDB
"Competitor just announced pricing — get it into institutional memory in 30 seconds."
HydraDB: Articles, tweets (free oEmbed), and YouTube transcripts ingest via connectors → add_memory(infer=true). No auth setup for public URLs.
  • Competitive intel: PMM pastes competitor blog URL — team queries it next week via search
  • Research: Cron-friendly: drop URLs in a folder + script loop, or use hydrabrain web for demos
  • Sales: Push prospect's latest funding announcement before outreach
  • All-hands: Push recorded talk transcript — employees chat with cited answers
One-shot or scripted push
hydrabrain read https://example.com/competitor-pricing-update
hydrabrain web --open   # paste links in browser UI

hydrabrain status

HydraDB
"Is the company brain healthy — and are crons actually running?"
HydraDB: Live tenant + memory count from API — no local DB to corrupt. doctor flags missing cron jobs and suggests fixes.
  • Ops: Daily health cron → pipe to Datadog/PagerDuty if memory count flatlines
  • IT: Verify HydraDB key + tenant before rolling out MCP to 50 agents
  • Post-incident: Confirm sync cron resumed after outage
Schedule (daily 6 AM)
hydrabrain cron add "0 6 * * *" "hydrabrain status >> ~/brain-health.log"

gbrain autopilot

Push ingbrain TS
"We run the full gbrain stack on Postgres — keep the git brain synced without babysitting."
vs HydraDB: Autopilot is the upstream TS daemon (sync → extract → embed). HydraBrain teams skip Postgres entirely — use hydrabrain sync crons instead. Same push intent, different backend.
  • Self-hosted brain: One install: gbrain autopilot --install → launchd/systemd/crontab
  • Git-backed wiki: Brain repo stays embedded and extracted on interval
  • Pair with dream: Add nightly gbrain dream for entity enrichment
Install once
gbrain autopilot --install
0 2 * * * gbrain dream >> ~/.gbrain/dream.log 2>&1

gbrain dream (nightly)

Push ingbrain TS
"Agents talked to 50 customers today — nothing should be lost by morning."
HydraDB equivalent: No dream cycle ported yet. Use capture during sessions + sync crons + briefing for overnight synthesis. Upstream gbrain runs 8-phase dream on PGLite/Postgres.
  • gbrain shops: Nightly entity sweep, citation fixes, pattern consolidation
  • HydraBrain shops: Nightly briefing + aggressive capture via agent MCP
  • Both: The brain compounds because push jobs run while employees sleep
gbrain only (2 AM)
0 2 * * * gbrain dream >> ~/.gbrain/dream.log 2>&1
# HydraBrain substitute:
0 2 * * * hydrabrain briefing >> ~/briefings/overnight.md

graph (MCP pull)

Pull outHydraDB
"Who connects us to widget-co — and through which investors?"
HydraDB: Native graph from infer=true on every push — no hand-rolled extractor. Relational queries that flat search misses become first-class.

Agent or CLI
